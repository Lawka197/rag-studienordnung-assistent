# Arbeitsjournal - 17-03-2026

## Ziel des Tages:
Einrichtung der Entwicklungsumgebungs und saubere Verbindung zwischen lokalem Projekt und GitHub-Repository.

## Erledigte Aufgaben:
- Lokales Projekt in PyCharm geöffnet
- Git-Repository initialisiert
- Remote Repository auf GitHub verbunden
- Konflikt mit bereits vorhandenem Remote-Inhalt (README, .gitignore) gelöst
- Lokales Repository mit Remote-Branch `main` verknüpft
- Unerwünschte Dateien wie `.idea` und `.DS_Store` aus dem Tracking entfernt
- Virtuelle Umgebung `.venv` erstellt und aktiviert
- Erste Python-Pakete installiert: `pypdf`, `python-dotenv`
- `requirements.txt` erzeugt

## Probleme
- Beim ersten `git push` wurde der Push abgelehnt, da das Remote-Repository bereits Inhalte hatte
- Zunächst wurden unerwünschte Dateien wie `.idea` und `.DS_Store` mitcommittet

## Lösungen
- Remote-Branch mit lokalem Branch verbunden und mit `git pull origin main --allow-unrelated-histories --no-rebase` zusammengeführt
- `.gitignore` genutzt und bereits getrackte Dateien mit `git rm --cached` entfernt
- Arbeiten ab dann innerhalb der aktivierten virtuellen Umgebung

## Ergebnis
Die Entwicklungsumgebung ist nun sauber vorbereitet. Das Projekt ist mit GitHub verbunden, die virtuelle Umgebung funktioniert, und erste Abhängigkeiten sind installiert
